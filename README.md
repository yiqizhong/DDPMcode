# DDPMCODE

Product-UI page generation for Dell device-management screens, organized as a
**three-layer, category-isolated, slot-based** structure with **Devin skills** doing the
generation.

## Three layers

- **Design tokens** (`shared/tokens.css`) — shared visual constants; rarely change.
- **Category template** — framework **skills** at the repo-root `.agents/skills/`, namespaced
  `headset-*` (each holds a slot-based template asset), plus the category stylesheet
  `headset/headset.css`. Generation goes through the skills, not hand-written HTML.
- **Model data** (`headset/models/<MODEL>/`) — one folder per concrete model: its manifests
  (清单) plus the generated, content-baked pages.

Differences between products live in the model manifest, never in the template. Variant and
list content is generated by presence/absence — no CSS-hidden, pre-embedded variants.

## Skill location

Skills live at the repo-root `.agents/skills/` (where Devin discovers them — it does not scan
nested dirs) and are namespaced `headset-…` to keep category isolation. This reconciles
methodology §9.5 (which nests skills per category) with Devin's root-only skill discovery —
the isolation intent is preserved via naming; only the folder location differs. See
`AGENTS.md`.

## Pilot

`headset/` is the current pilot category. It has the framework skills (`headset-gen-homepage`,
`headset-gen-subpage`, `headset-function`), layout, and category rules, but **no model
data yet** and **no dedicated `headset-function-<id>` skills or function snapshots yet** — those
grow from real manifests (methodology §9.4). Add models under `headset/models/` (see
`headset/models/README.md`). Other device categories get their own isolated folder later,
sharing only `shared/tokens.css`.

## Layout

```
.agents/skills/                       # repo ROOT — Devin discovers skills here (folders with SKILL.md)
  headset-gen-homepage/
    SKILL.md
    templates/home-frame.html         # slot-based home-page frame (copy + fill)
  headset-gen-subpage/
    SKILL.md
    templates/subpage-frame.html      # slot-based sub-page frame (holds any sub-page)
    templates/functions/              # sub-page bespoke-function snapshot registry
  headset-function/
    SKILL.md
    templates/function-frame.html     # function module generator (no-snapshot path; copy + fill)
  headset-shared/                     # NOT a skill — snippets shared by both gen skills:
    connection/  icons/  feature-button.html
shared/
  tokens.css                          # design-token layer
headset/
  AGENTS.md                           # category map: routing to skills + non-negotiables
  headset.css                         # category layout (references tokens via var())
  models/
    README.md                         # how to add a model (none yet)
AGENTS.md                             # cross-category map
```

See `AGENTS.md` for the cross-category map and `headset/AGENTS.md` for category routing.
