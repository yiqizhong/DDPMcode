从「产品需求」到「HTML 页面」的生成流程

一份耳机配套 App 的 UI，是怎样由一句产品需求自动生成的。

核心思路：manifest 是唯一事实源；除第 ① 步外全是确定性脚本 + 机械闸门；任何机械错误能自动纠正、重跑。

需要智能判断（LLM / 人） 确定性 · 机械（脚本 / 闸门） 🛠 Skill　＝ 该步用到的系统 skill
📄 用户输入：Product Requirement 产出 → requirements.md　（模板：设备身份 · 功能清单 · 功能描述 · 引导步骤）
↓
1作者化 ManifestLLM · 智能判断

读懂散文需求，做 authoring 决策：给每个功能选控件类型（toggle / segmented / slider / dropdown / preset-grid）、选项、默认值、条件显隐（reveals 显隐 · dependents 置灰），把判断冻结成结构化数据。这是全流程唯一需要理解的一步。

🛠 Skill headset-gen-homepage（首页 manifest） headset-gen-subpage（子页 manifest + 控件选择表） shared-gen-walkthrough（引导 manifest）
读入 requirements.mdheadset/AGENTS.md（控件选择决策表）各 SKILL.md schemaarchetypes.py（合法控件目录）
产出 home.manifest<功能>.manifest ×Nwalkthrough.manifestcoverage.md（需求↔manifest 原子对照表）
↓
2校验 Validate机械闸门

对照 schema 机械检查每份 manifest：未知字段、控件↔选项数规则、label 与卡标题重复、slider 数值范围、图片路径、恰好一个默认选中、嵌套深度…… 任一违规即 HALT 并指出位置。

🛠 Skill headset-gen-subpage　validate-manifest.py（子页） · validate-home.py（首页） · archetypes.py shared-gen-walkthrough　validate-walkthrough.py
读入home.manifest（首页）子页 *.manifestwalkthrough.manifest
产出通过 ✓ ／ HALT（附违规项）
↓
⚙ 机械错 → 自动改 manifest → 重渲
3渲染 Render确定性

把组件 snippet / 整卡 snapshot 按 manifest 复制进页面骨架、填入数值、拼出整页 —— 零推断、零手写，同样输入永远产出同样 HTML。

🛠 Skill headset-gen-subpage（render-model.py 编排 → render-home / render-subpage / render-content） headset-gen-homepage（首页骨架） headset-function（function-frame 卡壳） headset-shared（toggle/slider/segmented… 组件 · 图标 · 连接块） shared-gen-walkthrough（render-walkthrough.py）

4验证 Verify（全模型闸门）机械 · pre-commit + CI

对每个 model 跑全套机械检查：drift（磁盘 HTML 必须 == 渲染器输出，抓"有人手改产物"）· CSS 类必须真实存在 · 需求覆盖（功能 / 设备身份）· coverage 原子逐条比对。挂在 git pre-commit + CI 上，不绿不放行。

🛠 Skill headset-gen-subpage（verify-all.sh 编排 → verify-model · check-css-classes · check-requirements-coverage · check-coverage-atoms）

⚙ 机械错 → 自动修

未知 key · 错控件 · drift · 缺 CSS 类… 自动改 manifest 源头，重渲后回到 ③/④，循环到绿。

🛠 本系统的 5 个 Skill

流程里的每一步都由这些 skill 承担 —— 它们是系统的"工具箱"。

headset-gen-subpage系统核心：子页框架 + 全部校验 / 渲染 / 验证 / 自纠脚本（validate · render · verify · coverage · regen）都在这里。
headset-gen-homepage首页（设备身份 · 电量 · 连接 · 功能入口）的 schema 契约 + 骨架模板；它只含 SKILL.md + templates，首页的校验（validate-home.py）与渲染（render-home.py）实现都在 headset-gen-subpage 引擎里。
headset-function功能卡的卡壳（function-frame）；遇到未知功能时就地拼装、复用后沉淀成 snapshot。
headset-shared可复用的组件 snippet（toggle / slider / segmented / dropdown / preset-grid…）、图标、连接块、CSS。
shared-gen-walkthrough引导页（onboarding 多步走查）的 schema 校验与渲染。

manifest = 唯一事实源　·　产物可从 manifest 完全重生成，禁止手改　·　只有"要猜意图"才需要人
